About Dataset

  • Anonymized credit card transactions labeled as fraudulent or genuine
  • Source: [Kaggle] : https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud

The dataset contains transactions made by credit cards in September 2013 by European cardholders. This dataset presents transactions that occurred in two days, where we have 492 frauds out of 284,807 transactions. The dataset is highly unbalanced, the positive class (frauds) account for 0.172% of all transactions.

  • Features V1, V2, V28 are the principal components obtained with PCA,
  • The only features which have not been transformed with PCA are Time and Amount.
  • Feature Time contains the seconds elapsed between each transaction and the first transaction in the dataset.
  • The feature Amount is the transaction Amount, this feature can be used for example-dependant cost-sensitive learning.
  • Feature Class is the response variable and it takes value 1 in case of fraud and 0 otherwise

2. Random Forest Model

2.1 Data prepatation

  • First load the dataset and convert the target variable to a factor.
  • Second split the dataset into training and testing sets.
  • I also double-check the distribution of the target variable in the training and testing sets.
#import libraries
cc <- read.csv("creditcard.csv", header = TRUE)
cc$Class <- as.factor(cc$Class)

set.seed(456)
train <- sample(nrow(cc), 0.7*nrow(cc))
cc.train <- cc[train,]
cc.test <- cc[-train,]

table(cc.train$Class)
## 
##      0      1 
## 199006    358
table(cc.test$Class)
## 
##     0     1 
## 85309   134

2.2 Random Forest Model Building

  • Build the random forest model using the randomForest package
  • Set the seed to ensure reproducibility and use na.roughfix to impute missing values
  • Set importance to TRUE to get variable importance measures.
library(randomForest)
set.seed(123)
cc.forest <- randomForest(Class ~ ., 
                          data = cc.train, 
                          na.action = na.roughfix, 
                          importance = TRUE)
cc.forest

  • Based on the confusion matrix generated by the model evaluation, we can see that the model performs well in predicting the majority class (0, non-fraudulent transactions) with a high accuracy of 99.99992%
  • I can use area under the ROC curve (AUC-ROC) or area under the precision-recall curve (AUC-PR), which take into account the performance across all possible thresholds and are less affected by class imbalance. *note that accuracy, precision, recall, and F1-score metrics may not be suitable for imbalanced datasets

2.3 Model Evaluation

  • Make predictions on the test set using the predict function
  • Calculate the accuracy of the model and generate a confusion matrix.
  • Plot the variable importance measures
  • Calculate the ROC-AUC and plot the ROC curve.
cc.forest.pred <- predict(cc.forest, newdata = cc.test, type = "response")
table(cc.test$Class, cc.forest.pred, dnn = c("Actual", "Predicted"))
##       Predicted

mean(cc.test$Class == cc.forest.pred, na.rm = TRUE)
## [1] 0.9995553
  • Random forest model is performing very well with an OOB estimate of error rate of only 0.05% and a test set accuracy of 99.96%.
varImpPlot(cc.forest, 
           pch = 21, 
           color = "black", 
           bg = "red", 
           main = "Variable Importance for Credit Card Fraud")

- The most important variable is V14, followed by V17, V12, V11 and so on

cc.forest.pred <- predict(cc.forest, newdata = cc.test, type = "class")
cc.forest.pred.num <- as.numeric(as.character(cc.forest.pred))
roc_obj <- roc(cc.test$Class, cc.forest.pred.num)
auc <- auc(roc_obj)
cat("ROC-AUC:", auc, "\n")
## ROC-AUC: 0.8768364
plot(roc_obj, main = "ROC curve for Credit Card Fraud Random Forest Model")

- The ROC-AUC score for the model is 0.8768, which is relatively good. The ROC curve shows the trade-off between sensitivity and specificity for different thresholds.

2.4 Conclusion

  • The Random Forest model achieved a high accuracy and ROC-AUC score, indicating that it is a good model for detecting credit card fraud.
  • To further improve the model, tune the hyperparameters of the random forest model, such as the number of trees and the number of variables tried at each split. This can be done using cross-validation to find the optimal values.
